# Gymnastics Meet Manager

---------------


## What is currently implemented

### Backend API (Running)

* RESTful route structure

* Dependency-injected database sessions

* Pydantic request/response models

* Basic authentication scaffolding

* Error handling with HTTP exceptions

### Database Architecture
* Designed relational schema for:
    
    * Meets

    * Gymnasts

    * Directors

    * Judges

    * Scores

* Foreign key relationships properly structured

* Normalized data model

* SQLAlchemy ORM models implemented

---------

## Core Features Working
* Create meet

* Register gymnast

* Create score entry

* Prevent duplicate score entries

* Query meet and athlete data

## Why This Project?
This application is being built to modernize gymnastics meet management workflows, replacing manual processes and outdated scoring systems with a scalable, API-driven architecture.

The focus is on:

* Clean backend design

* Proper database relationships

* Role-based access control

* Real-world workflow modeling
